## Optimization Targets

### Target A: Defer _build_all_vehicle_log_caches (Bob, ~0.5s saved)
**Key insight**: `basic_analysis()` → `od_analysis()` does NOT access vehicle logs in C++ mode.
It uses `veh.distance_traveled`, `veh.state`, `veh.travel_time` — all direct C++ attributes.
`_build_vehicles_enter_log` uses `build_enter_log_data()`, not log caches.

**Solution**: Remove `_build_all_vehicle_log_caches()` from `simulation_terminated()`.
The existing `_ensure_log_raw()` mechanism already triggers lazy fetch when any vehicle log is accessed.

**Files**: uxsim_cpp_wrapper.py
**Risk**: Low — the lazy mechanism is already in place and tested.

### Target B: Interleaved LogEntry struct (Alice, ~0.3-0.5s saved from log_data)
**Problem**: log_data() does 5 push_backs to 5 separate vectors per timestep per vehicle.
5 separate write streams cause L2/L3 cache pressure.

**Solution**: Replace 5 per-vehicle log vectors with one:
```cpp
struct LogEntry {
    double t;     // 8
    double x;     // 8
    double v;     // 8
    int state;    // 4
    int link;     // 4
    int lane;     // 4
    int pad;      // 4 (to 40 bytes)
};
vector<LogEntry> log_entries;
```
Single push_back per timestep. Cache-line friendly writes.

**Impact on build_all_vehicle_logs_flat_full**: Must transpose from AoS to SoA.
But sequential read from interleaved + sequential write to separate arrays is efficient.

**Files**: traffi.h, traffi.cpp, bindings.cpp

### Target C: memcpy in build_all_vehicle_logs_flat_full (Alice, ~0.1-0.2s)
With separate vectors (current), use std::memcpy for bulk copy per vehicle:
```cpp
std::memcpy(&fl.log_t[base], v->log_t.data(), v->log_size * sizeof(double));
```
Replaces element-by-element copy loop. Compiler may already optimize this with -O3,
but explicit memcpy ensures it.

If Target B (interleaved) is implemented, this becomes a transpose operation instead.

## Priority and Assignment

1. **Target A** (Bob, %3): Remove _build_all_vehicle_log_caches from simulation_terminated. Simplest, biggest impact per effort. ~0.5s.
2. **Target C** (Alice, %1): memcpy optimization in build_all_vehicle_logs_flat_full. Quick win. ~0.1-0.2s.
3. **Target B** (Alice, %1): If time permits, interleaved LogEntry struct. Higher risk/reward.
